 Question: What are system pods in AKS?

Answer:

System pods are Kubernetes infrastructure components like CoreDNS, kube-proxy, and metrics-server that run in the kube-system namespace and maintain cluster networking, DNS, and monitoring. In AKS, these pods typically run on system node pools to ensure cluster stability.


System Node Pool (3 nodes)
   ├── CoreDNS
   ├── kube-proxy
   ├── metrics-server
   └── Azure storage drivers

User Node Pool (5 nodes)
   ├── frontend service
   ├── backend API
   ├── worker jobs
   └── databases

question difference docker and kubernates
ans- Docker is a containerization platform that allows you to create, deploy, and run applications in containers. 
It provides a way to package applications and their dependencies into a single unit that can run
consistently across different environments. 
Kubernetes, on the other hand, is a container orchestration platform that manages the deployment, scaling, 
and operation of containerized applications across a cluster of machines.

question - what is  architectural components of kubernates
ans- The architectural components of Kubernetes include:
1. Master Node: The control plane of the Kubernetes cluster that manages the cluster and makes global decisions about the cluster.
2. Worker Nodes: The nodes that run the containerized applications and are managed by the master node.
3. API Server: The component that exposes the Kubernetes API and serves as the entry point for all administrative tasks.
4. etcd: A distributed key-value store that stores the cluster's state and configuration data.
5. Controller Manager: The component that runs controllers to manage the desired state of the cluster and ensure that the cluster is running as expected.
6. Scheduler: The component that assigns workloads to worker nodes based on resource availability and other constraints.
7. Kubelet: The agent that runs on each worker node and ensures that the containers are running in a
pod and reports the status back to the master node.
8. Kube-proxy: The component that runs on each worker node and manages network communication between pods and services, providing load balancing and service discovery. 
9. Container Runtime: The software that runs and manages the containers on each worker node, such as Docker or containerd.
10. cloud controller manager: The component that manages cloud-specific resources and integrates with cloud providers to manage resources like load balancers, storage, and networking.

uestion- what is service in k8s
ans - A service in Kubernetes is an abstraction that defines a logical set of pods and a policy
by which to access them. It provides a stable IP address and DNS name for a set of pods, allowing for load balancing 
and service discovery within the cluster. Services enable communication between different components of 
an application and can also expose applications to external traffic.


Question - how many services in k8s
ans - there are several types of services in Kubernetes, including:
1. ClusterIP: Exposes the service on a cluster-internal IP. This is the default type and is only accessible within the cluster.
2. NodePort: Exposes the service on each node's IP at a static port. This allows external traffic to access the service using <NodeIP>:<NodePort>.
3. LoadBalancer: Provisions an external load balancer (if supported by the cloud provider) and assigns a public IP to the service, allowing external traffic to access the service.
4. ExternalName: Maps the service to an external DNS name, allowing access to services outside the cluster using a CNAME record.
5. Headless Service: A service without a cluster IP, used for stateful applications where clients need to connect directly to individual pods.

question- when these services are used in k8s cluster
ans- ClusterIP is used for internal communication between services within the cluster. 
NodePort is used when you want to expose a service to external traffic without using a load balancer.
LoadBalancer is used when you want to expose a service to external traffic and your cloud provider supports automatic load balancer provisioning.
ExternalName is used when you want to access an external service from within the cluster using a DNS name.
Headless Service is used for stateful applications where clients need to connect directly to individual pods, 
such as databases or stateful applications that require stable network identities.

question - what is event controller in k8s
ans- The Event Controller in Kubernetes is responsible for monitoring and managing events that occur within the cluster.
   It collects and processes events generated by various components of the cluster, such as pods, nodes, and services.

question - what are ingress controller
ans- Ingress controllers are responsible for managing and routing external traffic to services within a Kubernetes cluster.
   They provide a way to expose HTTP and HTTPS routes from outside the cluster to services within the cluster. 
   Ingress controllers typically use rules defined in Ingress resources to determine how to route traffic to the 
   appropriate services based on hostnames, paths, and other criteria. 
   They can also provide features like SSL termination, load balancing, and authentication for incoming traffic.
   It is providing the single entry point for all incoming traffic to the cluster and can route requests to 
   different services based on the defined rules in the Ingress resource.
   It is providing Enterprise level features like SSL termination, load balancing, and authentication for incoming traffic to the cluster 
   which is not provided by the default Kubernetes services like ClusterIP, NodePort, or LoadBalancer.
